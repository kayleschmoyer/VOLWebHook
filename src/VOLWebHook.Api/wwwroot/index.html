Quick Start

1

Send a Webhook

POST any JSON payload to the webhook endpoint:

  -H "Content-Type: application/json" \
  -d '{"event": "test", "data": {"message": "Hello World"}}'

Help & Documentation

Everything you need to know about using VOLWebHook

What is VOLWebHook?

VOLWebHook is a production-ready webhook capture service that acts as a universal endpoint for receiving webhook events from any external vendor or service. Think of it as a secure mailbox for your application's incoming webhook data.

When external services (like payment processors, CRM systems, or notification services) need to send real-time updates to your application, they use webhooks. VOLWebHook receives these webhooks, validates them, logs them, and stores them for later processing.

API Reference

POST /webhook

Receives webhook payloads from external services. Accepts any valid JSON body. Returns a confirmation with a unique request ID.

GET /health

Returns the service health status including uptime, environment, and security feature status.

GET /api/ui/webhooks

Returns a list of captured webhooks. Supports ?limit parameter for pagination.

GET /api/ui/webhooks/{id}

Returns full details of a specific webhook including headers and body.

GET /api/ui/stats

Returns webhook statistics including total count, today's count, and average payload size.

Go to the Configuration page and enable "IP Allowlist" under Security Settings. You can then add individual IP addresses or CIDR ranges (like 10.0.0.0/8) to the allowlist. Only requests from these IPs will be accepted.

By default, VOLWebHook is configured to always return HTTP 200 OK, even if there's an internal error or validation failure. This prevents external services from retrying webhooks unnecessarily. The actual error is logged for debugging.

The default retention period is 30 days. You can configure this in the Configuration page under "Retention Days". Webhooks older than this period are automatically cleaned up.
